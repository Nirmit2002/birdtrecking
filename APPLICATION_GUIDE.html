What Is This

A GPS bird tracker, visualised as a web app

Scientists fit White Storks with tiny GPS tags. Every few hours the tag records the bird's exact position, speed, altitude, battery level — and sends it back over mobile network. This application takes that raw data, processes it, and displays it as an interactive map with stats and charts that anyone can explore in a browser.

Think of it like Google Maps — but instead of tracking a car, it's tracking three individual storks across the Portuguese landscape in May 2022. The app lets you see where each bird flew, where it rested (stopovers), how fast it moved, and how high it climbed. It also has user login, an admin panel, and the ability to upload new data files.

In one line: Upload a GPS spreadsheet → the app draws the bird's route on a map, calculates stats, and shows it all through a secure web interface.

Technology

The tech stack — what each tool is and exactly why it's here

Six technologies work together. Below is each one explained in plain English — what it is, and the specific reason it was chosen for this bird-tracking application (not a generic "it's popular" answer).

Python
Core Language
What it is
A programming language — the instructions the computer runs. The entire application is written in Python. Think of it as the "language" this app speaks, like English is the language of this document.
Why in THIS app
Python has the world's best scientific data libraries — pandas, geopandas, scipy, movingpandas. GPS coordinate analysis, distance calculation on a curved Earth, detecting stopovers — all of this is built into Python libraries and would take months to build from scratch in another language.
Flask
Web Framework
What it is
A tool that turns a Python script into a website. Without Flask, Python has no way to respond to a web browser. Flask listens on a port, receives requests ("give me the dashboard"), and sends back the right HTML or data.
Why in THIS app
Flask is lightweight — it doesn't force a heavy project structure. For an academic project with 19 specific routes (login, map, admin, 6 API endpoints), Flask gives total control without boilerplate overhead. It also has Flask-Login and Flask-SQLAlchemy built in, which handle authentication and the database connection.
MySQL
Database
What it is
A database — a structured filing cabinet that stores information the app needs to remember across sessions (users, passwords, uploaded files). Without it, logging in would be impossible — data would reset every time the app restarts.
Why in THIS app
The app needs to store user accounts securely (bcrypt-hashed passwords), track which files each user uploaded, and support two roles (admin vs researcher) with different access. MySQL stores this reliably. It pairs with SQLAlchemy so Python talks to it without raw SQL — models.py describes the tables as Python classes.
Pandas & GeoPandas
Data Processing
What it is
Pandas is Excel inside Python — reads spreadsheets, filters rows, calculates means. GeoPandas is the same but understands coordinates — it can measure real distances on the curved surface of the Earth, not just subtract numbers.
Why in THIS app
The raw stork data is an Excel file with 99 rows × 30 columns. Pandas reads and cleans it in two lines. GeoPandas groups fixes by bird, sorts by timestamp, and uses Haversine geometry to measure actual flight distances. Without this, calculating "Mineiro flew 2.72 km" would require writing the spherical geometry math from scratch.
GeoJSON
Map Format
What it is
A text file format for storing geographic data — points, lines, and shapes with coordinates. Not a library or language, just a standard format. A GeoJSON file is just JSON with coordinates in a specific structure that maps understand.
Why in THIS app
The frontend map needs to draw the bird tracks. GeoJSON is the universal language all JavaScript map libraries speak natively. preprocess.py converts the Excel GPS data into four GeoJSON files (tracks, stopovers, stats, timeline) — these are read by the browser to draw lines and markers without any further calculation.
HTML / CSS / JS
Frontend
What it is
The three languages every browser runs. HTML is the structure (the skeleton of each page), CSS is the styling (colours, layout), JavaScript is the behaviour (map interactions, chart updates, button clicks).
Why in THIS app
The map page fetches GeoJSON from Flask's API via JavaScript without reloading — users can click a bird name and the map updates instantly. app.js calls /api/tracks, /api/stats etc. and passes results to a map library. charts.js draws the speed and altitude timelines. Jinja2 (Flask's template engine) fills in dynamic values like the logged-in username.

Architecture

How all the pieces connect and talk to each other

The application has three layers — browser, server, and storage. Here is exactly how a request travels from the moment you open the app to the moment you see a bird's track on the map.

1
Browser sends a request
You open localhost:5000/map. The browser sends an HTTP request to the Flask server running on your computer.
2
Flask routes it
app.py sees the URL /map, checks you're logged in (Flask-Login), runs the map_view() function, and sends back the map.html template.
3
JS fetches the data
The map page loads. JavaScript calls /api/tracks. Flask reads tracks.geojson and returns it as JSON. JS draws the bird paths on the map.

Data Pipeline

From GPS tag on a bird's back to a dot on the map

Raw GPS data goes through five clear transformation steps before you see anything on screen. Each step is handled by a different part of the codebase.

1

GPS tag records a fix

Every 1–4 hours, the backpack-mounted GPS tag on the stork locks onto satellites and records 30 values: latitude, longitude, altitude above sea level, ground speed, heading (compass direction), battery %, satellite count, accelerometer readings (detects wing flaps vs resting), external temperature, and more. The bird feels nothing — the tag weighs under 25g.

2

Tag transmits data via mobile network → Movebank → Excel file

The tag sends each fix via GSM (mobile network) to the Movebank animal tracking database. Researchers export the data as data.xlsx — a spreadsheet with 99 rows (30 fixes per bird × 3 birds = 90 data rows + headers). This file is placed in the app's data/ folder.

3

preprocess.py reads, cleans, and transforms the data

When the app starts for the first time, preprocess.py runs automatically. It reads the Excel file with pandas, converts coordinates to a GeoPandas dataframe, groups fixes by bird name, sorts each group by timestamp, connects points into lines, detects stopovers (clusters where the bird barely moved), and calculates stats. It writes four output files to static/geojson/: tracks.geojson, stopovers.geojson, stats.json, timeline.json. These are reused on every subsequent run — the heavy calculation only happens once.

4

Flask API serves the processed files on demand

When a user opens the map page, the browser's JavaScript calls Flask API endpoints: /api/tracks, /api/stopovers, /api/stats, /api/timeline. Each endpoint simply opens the corresponding GeoJSON file and returns it as a JSON response. This separation means the browser only downloads the data it needs, when it needs it — not everything at once.

5

JavaScript draws it on the interactive map

The browser receives the GeoJSON and uses a JavaScript map library to render bird tracks as coloured lines, stopovers as clickable circle markers, and individual GPS fixes as smaller dots. charts.js renders speed and altitude over time as line charts. Everything updates dynamically — clicking a bird name filters the map to show only that bird without reloading the page.

What the App Can Do

Every feature, explained

Migration Map
Interactive map showing all three birds' GPS tracks as coloured lines, stopovers as circle markers, and individual GPS fixes as smaller dots. Click any element for details. Pan and zoom freely.
Dashboard
Per-bird stat cards showing total distance, average speed, max speed, max altitude, stopover count, dominant heading, and tracking date range — all calculated from the GPS data.
Timeline Charts
Line charts for ground speed and altitude over time for each bird. Shows when the bird was flying fast vs resting, and how altitude changed during the tracking period.
Secure Login
Registration and login with bcrypt-hashed passwords. Session management via Flask-Login. Unauthenticated users are redirected to the login page automatically — no page is accessible without a valid session.
Admin Panel
Admin-only page showing all registered users, their roles, and registration dates. Admins can change a user's role (admin ↔ researcher) or delete accounts. Protected by a custom @admin_required decorator.
Data Upload
Upload a new Excel GPS file (up to 50 MB). The server runs preprocess.py on the new file, overwrites the GeoJSON output files, and the map updates with the new data on next load.

Getting Started

How to run the app on Windows (step by step)

Five commands, run once. After that just the last command each time to start the app.

1

Clone the code from GitHub

git clone https://github.com/Nirmit2002/birdtrecking.git
cd birdtrecking
2

Install all Python packages

pip install -r requirements.txt
pip install pymysql cryptography
3

Add MySQL to PATH (PowerShell — do this each session)

$env:PATH += ";C:\Program Files\MySQL\MySQL Server 8.0\bin"
4

Create the database and import data (one time only)

mysql -u root -p -e "CREATE DATABASE stork_db; CREATE USER 'stork_user'@'localhost' IDENTIFIED BY 'stork_pass2024'; GRANT ALL PRIVILEGES ON stork_db.* TO 'stork_user'@'localhost'; FLUSH PRIVILEGES;"
Get-Content stork_db.sql | mysql -u root -p stork_db
5

Start the app (run this every time)

python backend/app.py

Open http://localhost:5000 in any browser.
